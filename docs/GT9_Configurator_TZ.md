# ТЗ: GT-9 Configurator v2.0

## 1. Общее описание

Десктопное приложение для настройки и диагностики устройств серии Geotek GT (GT-9, GT-1, GT-3, GT-7) через USB COM-порт. Замена существующего конфигуратора на C# WinForms.

### 1.1 Технологический стек

- **Платформа:** Electron
- **UI:** React 18+ с TypeScript
- **Стилизация:** Tailwind CSS
- **UI-библиотека:** shadcn/ui
- **Таблицы:** TanStack Table (виртуальный скроллинг для 20000+ записей)
- **Serial Port:** npm-пакет `serialport`
- **Сборка:** Vite + electron-builder
- **Файловая система:** Node.js fs (для шаблонов, импорта/экспорта)

### 1.2 Протокол связи

Все команды используют текстовый протокол через COM-порт:

```
Запрос:  $<PASSWORD>;<COMMAND>[;<PARAMS>]\r\n
Ответ:   $<COMMAND>;<DATA>\r\n
```

- Пароль по умолчанию: `1234`
- Разделитель полей: `;`
- Разделитель подполей: `,`
- Протокол данных: **только CTR** (DATA_PROTO_TYPE = 1)

### 1.3 Коды ответов

| Код | Значение |
|-----|----------|
| `OK` | Успешно |
| `CE` | Ошибка команды |
| `DE` | Ошибка данных / конец данных |
| `PE` | Ошибка пароля |
| `FE` | Ошибка формата |

### 1.4 Типы устройств

| Код | Устройство |
|-----|-----------|
| 10 | GT-3 |
| 20 | GT-9 R4 (F1) |
| 21 | GT-9 R5 (F1) |
| 22 | GT-9 R6 (F1) |
| 23 | GT-9 R5 (F1) |
| 24 | GT-9 R6 (F4) |
| 25 | GT-1 |
| 26 | GT-9 R8 (F4) |
| 90 | GT-7 (F1) |

## 2. Общая архитектура UI

### 2.1 Верхняя панель (toolbar)

- **COM-порт:** Выпадающий список доступных портов + кнопка обновления

- **Перезагрузить:** Отправка команды `$1234;RESET` (доступна только при подключении)

- **Шаблоны:** Кнопки «Загрузить шаблон» / «Сохранить шаблон»
- **Иконки:** Загрузка прошивки, открыть файл настроек, сохранить файл настроек

### 2.2 Боковое меню (навигация)

Вертикальный список вкладок, активная вкладка подсвечивается:

1. Статус
2. Диагностика
3. Настройка сервера
4. Протокол
5. Настройка WIFI
6. GPS/Глонасс
7. Входы / Выходы

9. FLS (LLS)

## 3. Система шаблонов

### 3.1 Формат файла шаблона

Файл шаблона — текстовый файл (`.gtcfg`) с "сырыми" командами, по одной на строку:

```
$1234;APN;internet.mts.ru;mts;mts
$1234;SERVER1;SET;0F;01;s4.geotek.online;5601;860264052879153;860264052879153;10;0;120;30;1;0;1
$1234;FILTER;1,300;1,15;1,2;1,60;1;120;120
$1234;MSENS;SET;1;6;1;127;0;1;0
$1234;TILT;SET;1;30
$1234;IN1;3;1;33000;3000;2000;0;1;0;5;0
...
```

### 3.2 Сохранить шаблон

При нажатии «Сохранить шаблон»:
1. Для каждой вкладки — собрать текущие значения полей
2. Сформировать команды записи
3. Записать в файл `.gtcfg`
4. Диалог выбора пути сохранения

### 3.3 Загрузить шаблон

При нажатии «Загрузить шаблон»:
1. Диалог выбора файла `.gtcfg`
2. Прочитать файл построчно
3. Отправить каждую команду последовательно в COM-порт
4. Дождаться ответа перед отправкой следующей
5. Прогресс-бар с количеством обработанных команд
6. Лог результатов (OK / ошибки)

### 3.4 Замена пароля и IMEI в шаблоне

При загрузке шаблона:
- Пароль в командах (`$1234;`) заменяется на текущий пароль из toolbar
- IMEI в SERVER1 (поля LOGIN и PASS) заменяется на IMEI текущего устройства

### 4.1 Статус (read-only, обзорная панель)

**Режим работы:** Циклический опрос всех параметров с интервалом ~1-2 секунды.

#### Команды опроса (по кругу):

```
DEV → GSM → VER → DATE → REP → FUEL → IN → OUT → LLS1..LLS6 → ENCODER1;GET → ENCODER2;GET → TAGS
```

#### 4.1.1 Панель «Идентификация»

| Поле UI | Источник | Комментарий |
|---------|----------|-------------|
| Серия | `DEV` → DEV_ID | Read-only + кнопка копирования |
| IMEI | `GSM` → IMEI | Read-only + кнопка копирования |
| Прошивка | `VER` → FW_VERSION + RELEASE_DATE | Формат: "04.85 - 30.03.26" |
| Прибор | `VER` → HW_TYPE | Маппинг кода на название (24 → "GT-9 R6 (F4)") |

#### 4.1.2 Панель «Статус»

| Поле UI | Источник | Комментарий |
|---------|----------|-------------|
| Время | `DATE` → DATE + TIME | Формат: "DD-MM-YYYY HH:MM:SS" |
| Питание | `REP` → EXT_BAT | Значение в mV |
| Долг./шир. | `REP` → LATITUDE + LONGITUDE | "NA, NA" если нет фикса |
| Спутники | `REP` → SATINUSE | |
| GSM статус | `GSM` → STATUS | Маппинг кода на текст (TODO: таблица статусов) |
| GSM RSSI | `GSM` → RSSI | Пересчёт в % |
| WIFI статус | `REP` → WIFI_STATUS | Маппинг кода на текст |
| WIFI RSSI | `REP` → WIFI_RSSI | Значение в dBm |
| Внт. температура | `REP` → INT_TEMP | Значение в °C |

#### 4.1.3 Панель «ТРК/насосы»

| Поле UI | Источник |
|---------|----------|
| P1-P4 (красные/зелёные) | `FUEL` → FUEL1..FUEL4 (текущая доза) |
| C1-C4 | `FUEL` → TOTAL1..TOTAL4 (тотализатор) |

Цвет: красный при STATUS=-0001 (неактивен), зелёный при активной выдаче.

#### 4.1.4 Панель «Входа/Выходы»

| Поле UI | Источник |
|---------|----------|
| IN1-IN6 | `IN` → значения входов |
| E1, E2 | `ENCODER1;GET`, `ENCODER2;GET` → COUNTER_VALUE |
| OUT1-OUT4 | `OUT` → значения выходов |

**Интерактивные элементы:**
- Кнопки **ON / OFF** у каждого выхода: `$1234;OUTn;;1` (ON) / `$1234;OUTn;;0` (OFF)

#### 4.1.5 Панель «Датчики уровня»

Два столбца (Датчик 1 и Датчик 2), данные из `LLS1`-`LLS6`:

| Поле UI | Поле ответа LLSn |
|---------|------------------|
| Высота N, мм. | LENGTH (индекс [3] в live-ответе) |
| Объём N, лт. | CAPACITY (индекс [4]) |
| Температура N, C | TEMPERATURE (индекс [5]) |
| Плотность N, кг/м3 | DENSITY (индекс [6]) |

#### 4.1.6 Панель «Карты»

| Поле UI | Источник |
|---------|----------|
| Последняя | `REP` → последние 12 символов ответа (TAG_ID) |
| Память | `TAGS` → MEMORY |
| Лимит | `TAGS` → LIMIT |
| Добавлено | `TAGS` → ADDED |

**Интерактивные элементы:**
- Кнопка **add** у «Последняя»: добавляет последний приложенный ключ в память → `$1234;TAGS;ADD;TAG_ID`

### 4.4 Протокол

Управление полями в протоколе передачи данных. **Только протокол CTR** (DATA_PROTO_TYPE = 1).

#### 4.4.1 Чекбоксы — 5 колонок

**Статус:**
| Чекбокс | Tag ID |
|---------|--------|
| Статус прибора | 0x0E |
| Номер пакета | 0x0F |
| HDOP | 0x17 |
| Внешнее питание | 0x20 |
| Аккумулятор | 0x21 |
| Внут. температура | 0x24 |
| GSM LBS | 0x32 |
| GSM RSSI | 0x31 |

**Входа:**
| Чекбокс | Tag ID |
|---------|--------|
| IN 1-8 | 0x50-0x57 |

**Выходы:**
| Чекбокс | Tag ID |
|---------|--------|
| OUT 1-4 | 0x60-0x63 |

**ДУТ:**
| Чекбокс | Tag ID |
|---------|--------|
| LLS 1-6 | 0x70-0x75 |

**ДУТ НПП:**
| Чекбокс | Tag ID |
|---------|--------|
| PMP 1-6 | 0x78-0x7D |

#### 4.4.2 Два способа записи

**Массовый** (PRSET):
```
Считать: PRSET20;GET + PRSET21;GET
Записать: PRSET20;SET;{32-hex-char} + PRSET21;SET;{32-hex-char}
```
- PRSET20 — байты 0-7
- PRSET21 — байты 8-15

**Штучный** (PROTOCOL):
```
Включить: PROTOCOL;SET;1;{ID1};{ID2};...
Выключить: PROTOCOL;RESET;1;{ID1};{ID2};...
```

#### Считать / Сохранить

- Считать: `PRSET20;GET` + `PRSET21;GET` → декодировать bitmask → установить чекбоксы
- Сохранить: собрать bitmask из чекбоксов → `PRSET20;SET;...` + `PRSET21;SET;...`

### 4.14 Ключи/метки

#### 4.14.1 Структура

**Одна большая таблица** с виртуальным скроллингом (до 20 000 записей).

**Колонки таблицы:**

| Колонка | Тип | Поле TAG |
|---------|-----|----------|
| № | number (read-only) | INDEX |
| Tag ID | text (12 hex) | TAG_ID |
| Лимит | number (0-9999) | PARAM1 |
| Параметр | number (hex bitmask) | PARAM2 |
| Пин код | number (0-9999) | PARAM3 |
| Действия | кнопки | Редактировать, Удалить |

#### 4.14.2 Функции

- **Inline редактирование** — клик на ячейку → редактирование → сохранение: `TAG;SET;TAG_ID;LIMIT;PARAM;PIN`
- **Удалить строку** — `TAG;DEL;TAG_ID`
- **Добавить ключ** — поле ввода TAG_ID + кнопка «Добавить»: `TAGS;ADD;TAG_ID`
- **Считать все** — цикл `TAG;GETI;0`..`TAG;GETI;N` (N из `TAGS` → ADDED). Прогресс-бар обязателен.
- **Импорт из файла** — загрузка ключей из CSV/TXT файла
- **Экспорт в файл** — сохранение ключей в CSV/TXT файл

#### 4.14.3 Верхняя панель

- **Последний ключ** — отображение TAG_ID + кнопки «Считать» (прочитать параметры) и «Добавить» (добавить в память)

#### Виртуальный скроллинг

Обязательно использование TanStack Table с виртуализацией — нельзя рендерить 20000 строк одновременно.

## 5. Замечания для обновления commands.md

Список расхождений, обнаруженных при анализе вкладок:

1. **GSM** — ответ содержит 16 полей, описано 10. Последние 6 полей (таймауты) не документированы.
2. **REP** — последнее поле (TAG_ID последней карты) не описано.
3. **GSM STATUS** — нет таблицы маппинга status-кодов на текстовые значения.
4. **INn** — ответ при чтении содержит 12 полей, описано 10. Последние 2 не документированы.
5. **LLSn;GET** — ответ содержит 8 полей, описано 7. Восьмое поле не документировано.
6. **TILT;GET** — ответ содержит 3 поля, описано 2. Третье поле (текущий угол?) не документировано.
7. **DEVICE_TYPE для RS-портов** — описано 7 типов, фактически 40+. Полный список в разделе 4.8.
8. **PUMPn** — описано 10 полей, фактически 17. Поля 2ND_OUT, 2ND_START, 2ND_STOP, RFID_MODE, ROUND, PRICE не документированы.
9. **PUMPSEC** — ответ при чтении содержит 10 полей, описано 9. Десятое поле не документировано.
10. **PRINTER;GET** — ответ содержит 4 поля, описано 3. Четвёртое поле не документировано.
11. **PRNTN/PRNTP/PRNTW** — нет описания формата кодирования (UTF-16LE).
12. **PUMPFRMT** — нет описания значений VALUE_FMT, TOTAL_FMT, LIMIT_FMT, LIMIT_LEN.
13. **SERVER1** — поле IP поддерживает hostname, а не только IP. Описание нужно обновить.
14. **PRSET** — нет явного описания разницы PRSET10/11 (IPS) vs PRSET20/21 (CTR).
